jawn vault

secure credential proxy for the houseofjawn network

rust phase 2 / security hardening MIT

what it does

fast cache
sub-5ms retrieval vs ~128ms raw GPG decryption. in-memory, TTL-based, secrets zeroed on eviction.
unix socket API
no network exposure. kernel-enforced access control via SO_PEERCRED.
audit log
full trail of who accessed what credential, when. SQLite with configurable retention.
memory safety
secrets zeroed on drop. uses the secrecy and zeroize crates. no accidental log exposure.
token auth
bearer tokens with path-based permissions. stored as SHA-256 hashes, never plaintext.
pass backend
wraps the existing pass (GPG) store. no migration needed — same secrets, faster access.

quick start

# build
cargo build --release

# start the daemon
./target/release/jawn-vault

# create a client token (path pattern, read-only)
./target/release/vault-cli token create houseofjawn-bot \
  --grant 'claude/**' --permission read

# use it
export VAULT_TOKEN='vault_abc123...'
./target/release/vault-cli get claude/api/anthropic
